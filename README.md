# 📘 Codesaga Day 44 - String Methods

## 🎯 Topic Covered

- upper()
- lower()
- replace()


## 📖 What I Learned

- upper() creates an uppercase version of a string.
- lower() creates a lowercase version of a string.
- replace() creates a new string with replaced text.
- Strings are immutable.
- Original strings stay safe and unchanged.
- Python creates a new string instead of modifying the original one.
- Case sensitivity matters in searching and comparison.
- Real systems often keep an original version and a search-friendly version of data.


## ⚠️ Mistakes I Made

- I thought replace() changes the original string.
- I mixed "new string creation" with "original string modification".
- I focused on syntax first and missed the design thinking behind string methods.
